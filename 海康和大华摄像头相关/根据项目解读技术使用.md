# 名词解释

## flv.js 播放的视频 项目案例

# 一、整体技术栈（一眼看懂）

```
前端：HTML5 + jQuery + axios + flv.js
流媒体：HTTP-FLV 直播流
编码：H.264 / H.265 都支持（看后端输出）
协议：FLV over HTTP（低延迟直播）
场景：安防监控直播、摄像头实时预览
```

### 技术特点

- **纯前端解码、播放 FLV 直播流**
- 不需要 Flash、不需要插件
- 浏览器原生 HTML5 Video 渲染

## 2. 传输协议：HTTP-FLV

你的代码明确获取：

```
type=flv
```

所以播放地址是：

```
http://xxx/xxx.flv  播放地址后缀特点
```

### HTTP-FLV 特点

- 低延迟：**1~3 秒**
- 比 HLS (m3u8) 快 3~10 倍
- 适合监控、直播
- 依赖 flv.js

------

## 3. 编码格式：H.264 / H.265

代码**不限制编码**，只要流是 FLV 封装：

- 大多数监控默认：**H.264**
- 新设备 / 高清：**H.265 (HEVC)**
- flv.js 新版都支持

## 播放生命周期管理（专业）

```
btnPlay()  → 创建 → load → play
btnStop() → pause → unload → detach → destroy
```

# 套系统架构（你这套的完整流程）

```
摄像头 → 流媒体服务 → 输出 HTTP-FLV 流 →
前端登录获取 token → 获取播放URL →
flv.js 解码 → HTML5 Video 播放 → 自动追帧保持低延迟
```



### 最终极简总结

- **编码**：H.264 / H.265
- **传输**：HTTP-FLV（低延迟）
- **播放**：flv.js + HTML5 Video
- **核心特点**：低延迟、自动追帧、纯网页、云台控制
- **延迟**：1~3 秒
- **适用**：监控摄像头直播

// 创建 flv.js 播放器对象
	function createFlvPlayer(videoEle, url) {
        
		var flvPlayer = flvjs.createPlayer({
			type: 'flv',
			isLive: true,
			enableWorker: true,
			hasVideo: true,
			hasAudio: false,
			withCredentials: false,
			enableStashBuffer: false,
			stashInitialSize: 6,
			playbackRate: 1.0,
			url: url
		});
		flvPlayer.attachMediaElement(videoEle);
        
		return flvPlayer    
	}

// 开始播放URL
	function btnPlay() {
		console.log("开始播放URL...")
		var videoElement = document.getElementById('videodisplay');
		flvPlayer = createFlvPlayer(
			videoElement,
			playurl
			//'http://1*1.3*.**.234/flv?app=live&stream=4**&user=**&videoType=0'
		);
		flvPlayer.load();
		flvPlayer.play();
		
		// 定时追帧, 保持播放画面的实时性
		timerChaseFrame = setInterval(() => {
			if (!videoElement.buffered.length) {
				return;
			}
			const end = videoElement.buffered.end(0)
			const current = videoElement.currentTime
			const diff = end - current
			const diffCritical = 8 // 超过8秒以上就进行跳转
			const diffSpeedUp = 2  // 超过2秒以上则进行视频加速播放
			const maxPlaybackRate = 1.2 // 加速播放的速度
			let playbackRate = 1.0 // 播放速度
			if (diff > diffCritical) {
				videoElement.currentTime = end - 0.45 // 这里不跳到end，是因为如果buffer的增长速度相对较快的时候，会出现频繁追帧而导致明显的卡顿
				playbackRate = Math.max(1, Math.min(diffCritical, 16))
			} else if (diff > diffSpeedUp) {
				playbackRate = Math.max(1, Math.min(maxPlaybackRate, 16))
			}
			videoElement.playbackRate = playbackRate
			if (videoElement.paused) flvPlayer.play()
		}, 2000);
	}
	
	// 停止播放URL
	function btnStop() {
		clearInterval(timerChaseFrame);
		if (flvPlayer != null) {
			flvPlayer.pause();
			flvPlayer.unload();
			flvPlayer.detachMediaElement();
			flvPlayer.destroy();
			flvPlayer = null;
		}
	}

## HLS (m3u8)

**全称**：HTTP Live Streaming

**一句话解释**：**苹果推出的视频直播 / 点播协议，把视频切成小碎片播放**

- 本质：**视频传输协议**（怎么把视频从服务器发给播放器）

- 工作原理：

  1. 把视频切成 **1~10 秒的小文件（.ts）**
  2. 生成一个索引文件（.m3u8）
  3. 播放器按顺序下载小碎片，边下载边播放

- 优点：

  ✅ 全平台兼容（手机、浏览器、电视都支持）

  ✅ 网络不好时自动切换清晰度

  ✅ 穿透防火墙强（用普通 HTTP 协议）

- 缺点：**延迟较高**（通常 3~30 秒，不适合超低延迟直播）

**简单类比**：

HLS = 把一部电影剪成无数个 5 秒小片段，播放器按顺序播放，不用等整部电影下载完

**协议本质**：HTTP 切片流媒体，苹果标准

**播放工具**：原生浏览器 / 手机系统自带播放器、Video.js、CKPlayer 等

**视频播放方式**：视频切小分片 (.ts)，加载 m3u8 索引边下边播

✅ 优点：**兼容性天花板**、穿墙好、支持多清晰度自适应、无需额外插件

❌ 缺点：延迟高 (3~15s+)、直播实时性差

**地址后缀**：`.m3u8`

# 一、FLV（HTTP-FLV）特点

**你现在代码里正在用的：低延迟直播方案**

## 核心

- **封装格式**：FLV
- **传输协议**：HTTP-FLV
- **播放工具**：**flv.js**（浏览器必须用）
- **地址后缀**：`.flv`

## 优点

✅ **延迟极低**：1~3 秒（监控 / 直播首选）

✅ **流畅不卡顿**：流式实时传输

✅ **不累积延迟**：配合追帧永远实时

✅ **带宽占用小**

✅ **适合网页直播、摄像头监控**

## 缺点

❌ **iOS 浏览器不支持**（苹果 Safari 播不了）

❌ 依赖 flv.js 前端解码

❌ 兼容性不如 HLS 通用

## 适用场景

- 低延迟直播
- 网页监控
- PC 端管理平台
- 延迟要求 <3 秒
