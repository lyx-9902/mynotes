## linux类服务器简介

Ubuntu CentOS Debian Rocky Linux AlmaLinux RedHat(RHEL) Fedora 和linux区别

要搞懂这些名词和「Linux」的区别，核心结论先明确：**Linux 是「内核」（操作系统的核心核心），而 Ubuntu、CentOS 等都是基于 Linux 内核构建的「操作系统发行版」（完整可用的操作系统）** —— 类比来说，Linux 像「汽车发动机」，各发行版像「装配了该发动机的完整汽车」（外观、配置、用途不同，但核心动力都是 Linux）。

### 先理清两个关键概念

| 概念                 | 本质                                                         | 作用                                                         |
| -------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Linux**            | 操作系统内核（Kernel），仅提供硬件管理（CPU、内存、硬盘）、进程调度等最底层功能 | 是所有 Linux 发行版的「核心动力」，但单独用不了（没有图形界面、命令行工具等） |
| **发行版（Distro）** | 厂商 / 社区以 Linux 内核为基础，搭配「系统工具、软件库、桌面环境（可选）、包管理器」等，打包成的完整操作系统 | 开箱即用，满足用户日常办公、服务器部署、开发等实际需求       |

### 各发行版的核心区别（定位、用途、特点）

下面逐个拆解你提到的发行版，重点说它们的「定位差异」和「适用场景」，避免混淆：

| 发行版            | 核心定位                                                    | 基于 / 关联关系                                  | 关键特点（优点 / 缺点）                                      | 适用场景                                     |
| ----------------- | ----------------------------------------------------------- | ------------------------------------------------ | ------------------------------------------------------------ | -------------------------------------------- |
| **Ubuntu**        | 桌面 / 服务器通用，主打「易用性」，面向个人用户和中小企业   | 基于 Debian（简化版）                            | ✅ 生态丰富、软件多、社区支持强；❌ 服务器场景下稳定性略逊于 RHEL 系 | 个人桌面（新手首选）、开发机、小型服务器     |
| **Debian**        | 追求「稳定、自由、开源纯粹」，是很多发行版的「上游源头」    | 独立开发（无直接上游）                           | ✅ 稳定性极高、开源合规性强；❌ 软件版本偏旧（优先稳定）、配置略复杂 | 对稳定性要求高的服务器、开源项目测试环境     |
| **RedHat (RHEL)** | 企业级商业发行版，主打「稳定性、安全合规、官方技术支持」    | 独立商业开发（RHEL 系源头）                      | ✅ 长期支持（7-10 年）、企业级安全补丁、合规认证齐全；❌ 收费（需订阅支持） | 大型企业服务器、金融 / 政务等关键业务        |
| **CentOS**        | 曾是 RHEL 的「免费开源克隆版」（完全兼容 RHEL，无商业支持） | 原基于 RHEL，CentOS 8 后停更，改为 CentOS Stream | ✅ 原版本免费、兼容 RHEL；❌ 停更后无稳定版，Stream 是 RHEL 测试版（不适合生产） | 原 CentOS 7 仍广泛用于老服务器，新场景不推荐 |
| **Rocky Linux**   | 替代 CentOS 的「免费企业级发行版」，目标 1:1 兼容 RHEL      | 基于 RHEL 源码重建（CentOS 原团队主导）          | ✅ 免费、完全兼容 RHEL、长期支持、社区驱动；❌ 生态成熟度略逊于 RHEL/CentOS 老版本 | 企业服务器（CentOS 迁移首选）、生产环境      |
| **AlmaLinux**     | 另一款 CentOS 替代者，同样兼容 RHEL 的免费企业级发行版      | 基于 RHEL 源码重建（CloudLinux 公司主导）        | ✅ 免费、兼容 RHEL、商业公司支持（稳定性有保障）；❌ 与 Rocky Linux 功能高度重叠 | 企业服务器、生产环境（CentOS 迁移备选）      |
| **Fedora**        | RHEL 的「测试 / 预览版」，主打「前沿特性、新技术尝鲜」      | 由 RedHat 赞助（RHEL 上游）                      | ✅ 软件版本新、支持最新硬件 / 技术；❌ 生命周期短（仅 18 个月）、稳定性略差 | 开发者、技术尝鲜者、桌面用户（喜欢新功能）   |

在阿里云服务器中，公有 IP 和私有 IP 的用途有明显区别，具体如下：

### 公有 IP（公网 IP，如弹性公网 IP EIP）

- **对外服务入口**：是服务器在互联网上的 “门牌号”，用于外部用户（如互联网上的电脑、手机）访问服务器上的服务，例如部署的个人博客网站、Web 应用、API 接口等，用户通过浏览器输入公有 IP 即可访问网站。
- **远程管理服务器**：用户可以通过 SSH（Linux 系统）、RDP（Windows 系统）等协议，利用公有 IP 远程连接服务器，进行系统配置、软件安装、日志查看等管理操作。
- **对外通信**：使服务器能够访问互联网资源，比如下载软件包、更新系统等。

### 私有 IP（内网 IP）

- **内网资源通信**：用于阿里云同一私有网络（VPC）内的云服务器、数据库（如 RDS）、缓存（如 Redis）等资源之间的内部通信。例如，前端服务器（绑定公有 IP）可以通过私有 IP 访问内网的数据库服务器，保障数据库的安全性，因为私有 IP 无法直接从互联网访问。
- **降低成本**：内网之间的流量通常免费或成本极低，相比公网带宽费用更节省，适合大量数据在内部传输的场景，如分布式系统中多台服务器之间的数据同步。
- **安全隔离**：由于私有 IP 不暴露在互联网上，能有效避免外部网络攻击，提升内网资源的安全性，常用于部署敏感的后端服务，如数据库、核心业务逻辑服务器等。

### 一、先查：服务器操作系统版本（核心信息）

执行以下命令，直接获取系统发行版、版本号等关键信息：

bash

```bash
cat /etc/os-release
```

#### 输出示例（阿里云常见系统）：

bash

```bash
NAME="Alibaba Cloud Linux"  # 系统名称（阿里云自研Linux，最常用）
VERSION="3 (Soaring Falcon)" # 系统版本
ID="alinux"                  # 系统标识

ANSI_COLOR="0;31"
HOME_URL="https://www.aliyun.com/"
```

其他可能的输出（若你选了其他镜像）：

- 若显示 `NAME="Ubuntu"` → Ubuntu 系统；
- 若显示 `NAME="CentOS Linux"` → CentOS 系统；
- 若显示 `NAME="Debian"` → Debian 系统。

### 二、再查：服务器硬件配置（CPU / 内存 / 存储）

确认服务器的硬件规格（和你购买的套餐是否一致）：

#### 1. 查看 CPU 信息（核心数、架构）

bash

```bash
lscpu
```

关键看 `CPU(s):`（总核心数，如 2 核）、`Architecture:`（架构，如 x86_64）。

#### 2. 查看内存信息（总内存、可用内存）

bash

```bash
free -h  # -h 表示以人类可读单位（GB/MB）显示
```

输出示例：`Mem: 2.0Gi 1.2Gi 800Mi 0.0Ki 0.0Ki 800Mi` → 总内存 2GB。

#### 3. 查看存储信息（磁盘大小、类型）

bash

```bash
df -h  # 查看挂载的磁盘空间
```

关键看 `/` 根目录的 `Size`（如 40G）、`Filesystem`（如 `/dev/vda1`，对应阿里云 ESSD 磁盘）。

### 三、最后查：已安装的关键服务（判断镜像类型）

结合你之前关注的 “Docker 镜像” 或 “应用镜像”，查看是否预装了对应服务：

#### 1. 检查是否是 Docker 镜像（最常用）

执行以下命令，若有输出 Docker 版本，说明是 Docker 镜像（预装了 Docker 环境）：

bash

```bash
docker --version
```

输出示例：`Docker version 27.0.3, build 7d4bcd8` → 确认是 Docker 镜像。

#### 2. 检查是否是应用镜像（如 WordPress、宝塔面板）

- 若查 WordPress：检查是否有 Apache/Nginx + MySQL + PHP 环境：

  bash

  ```bash
  # 检查 Nginx/Apache 服务
  systemctl status nginx || systemctl status httpd
  # 检查 MySQL 服务
  systemctl status mysqld || systemctl status mariadb
  # 检查 PHP 版本
  php --version
  ```

  若以上服务均存在，大概率是 WordPress 镜像。

- 若查宝塔面板：检查是否有宝塔服务（bt）：

  bash

  ```bash
  bt status
  ```

  

  若输出 “宝塔面板已启动”，说明是宝塔镜像。

### 四、总结：快速判断流程

1. 用 `cat /etc/os-release` 确定 **系统版本**（如 Alibaba Cloud Linux 3）；
2. 用 `docker --version` 确认 **是否是 Docker 镜像**（最符合你之前的需求）；
3. 用 `lscpu`/`free -h`/`df -h` 验证 **硬件配置**（2 核 2G40G ESSD）是否和购买套餐一致。

## 解释一下 左侧的路径，和右侧的文件夹路径的区别

<img src=".\img\3.png" style="zoom: 80%;" />

在这张截图中，左侧的 “路径” 和右侧的 “文件夹路径” 是 **Linux 系统中两种不同层级的目录展示**，核心区别如下：

### 1. 左侧 “路径”：系统级存储资源的使用情况

左侧展示的是 **系统核心存储设备或特殊目录的 “可用 / 大小” 比例**，属于**系统资源层面的路径**，例如：

- `/dev`：系统设备目录（存放硬件设备的映射文件，如磁盘、网卡等）；
- `/dev/shm`：内存共享目录（基于内存的临时存储，用于进程间快速通信）；
- `/sys/fs/cgroup`：Linux 控制组目录（用于资源隔离和限制，如 CPU、内存的分配）；
- `/run/user/0`：root 用户的运行时临时目录（存放进程运行中的临时文件）。

这些路径是**系统底层的核心目录**，普通用户一般不会直接操作，主要用于系统级的资源管理和硬件交互。

### 2. 右侧 “文件夹路径”：服务器文件系统的目录结构

右侧展示的是 **Linux 服务器的标准文件系统目录结构**，属于**用户可操作的文件 / 文件夹路径**，例如：

- `/bin`：系统基础命令的存储目录（如 `ls`、`cd` 等命令）；
- `/home`：普通用户的主目录（每个用户的个人文件会存放在这里，如截图中的 `admin` 文件夹）；
- `/root`：root 管理员的主目录；
- `/etc`：系统配置文件目录（如网络配置、服务配置等）。

这些路径是**用户日常操作的核心区域**，用于存放程序、数据、配置文件等，是部署应用（如个人博客）、管理文件的主要操作对象。

### 总结：核心区别

| 维度     | 左侧 “路径”                        | 右侧 “文件夹路径”                |
| -------- | ---------------------------------- | -------------------------------- |
| 用途     | 系统资源监控（存储设备、临时目录） | 用户文件管理（程序、数据、配置） |
| 操作权限 | 多为系统级，普通用户无操作权限     | 普通用户可在自己目录下自由操作   |
| 典型场景 | 查看系统设备占用、内存共享区大小   | 部署博客程序、上传文章、改配置   |

简单来说，左侧是 “系统底层的资源目录”，用于监控系统运行状态；右侧是 “用户可操作的文件目录”，是你部署个人博客、管理文件的实际工作区域。

## 常用的shell命令

### 一、核心命令：查看当前所在文件夹（pwd）

要知道自己现在处于服务器的哪个目录（即 “当前文件夹”），执行以下命令：

```
pwd
```

- 命令含义：`pwd` = Print Working Directory（打印工作目录），直接输出当前所在文件夹的完整路径。

- 示例输出：

  bash

  ```bash
  /root  # 表示当前在 root 用户的主目录（默认登录后会进入这里）
  ```

### 二、核心命令：切换文件夹（cd）

`d` = Change Directory（切换目录），用于进入不同的文件夹，语法和常用场景如下：

#### 1. 基础用法：切换到指定文件夹

bash

```bash
cd 目标文件夹路径
```

- 示例 1：进入

  ```
  /home
  ```

  文件夹（绝对路径，从根目录

  ```
  /
  ```

  开始写）

  bash

  ```bash
  cd /home
  ```

- 示例 2：进入当前文件夹下的

  ```
  docker
  ```

  子文件夹（相对路径，基于当前所在目录）

  bash

  ```bash
  cd docker  # 前提是当前目录下有 docker 文件夹
  ```

#### 2. 常用快捷操作（必记！）

| 命令    | 作用说明                                                     |
| ------- | ------------------------------------------------------------ |
| `cd ~`  | 切换到当前用户的主目录（root 用户主目录是 `/root`，普通用户是 `/home/用户名`） |
| `cd ..` | 切换到 “上一级文件夹”（返回父目录），比如从 `/root/docker` 回到 `/root` |
| `cd .`  | 留在当前文件夹（无实际作用，仅了解语法）                     |
| `cd -`  | 切换到上一次所在的文件夹（类似 “返回上一步”）                |



## * Linux 系统中的用户身份

在 Linux 系统中，`root`（超级管理员）和普通用户是两种权限级别完全不同的用户身份，核心区别如下：

| **维度**     | **root 用户**                                                | **普通用户**                                                 |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **权限范围** | 拥有系统**全部权限**，可修改任何文件、安装 / 删除软件、重启系统等 | 权限受限，仅能操作自己目录下的文件，需 `sudo` 才能执行高权限操作 |
| **安全风险** | 操作失误可能直接导致系统崩溃（如误删系统文件）               | 风险低，即使操作失误也不会影响系统核心功能                   |
| **登录场景** | 仅在需要系统级管理时使用（如安装服务、修改系统配置）         | 日常操作（如写博客、编辑文档）的默认身份                     |
| **切换方式** | 普通用户可通过 `su - root`（输入 root 密码）或 `sudo -i`（输入当前用户密码）切换为 root | root 可通过 `su - 用户名` 切换为普通用户                     |

### 典型使用场景

- **用 root 的场景**：安装 Docker、配置服务器网络、修改系统级配置文件（如 `/etc` 下的文件）。
- **用普通用户的场景**：写博客文章、编辑个人代码、浏览文件等日常操作。

### 安全建议

- 日常操作**优先用普通用户**，避免误操作破坏系统；
- 如需执行高权限操作，建议用 `sudo 命令`（而非直接登录 root），例如 `sudo docker ps`（输入当前用户密码即可临时获得 root 权限执行命令）。

### *  查询当前系统有多少用户



![](.\img\4.png)

要确定当前系统中有多少用户，我们可以通过分析 `/etc/passwd` 文件和 `/home` 目录来区分**系统用户**和**可登录的普通用户**：

### 1. 系统用户（用于服务运行，不可登录）

通过 `cat /etc/passwd` 输出，这些用户的登录 Shell 多为 `/sbin/nologin`，是系统为了运行服务（如 `sshd`、`mysql`、`nginx` 等）自动创建的，例如：

- `bin`、`daemon`、`adm`、`lp`、`sync` 等（前 20 行左右均为系统用户）。

### 2. 可登录的普通用户（仅 1 个）

- 从 `/home` 目录的 `ls` 输出可见，只有 `admin` 文件夹，说明存在**普通用户 `admin`**；
- 结合 `/etc/passwd` 最后一行 `admin:x:1000:29:IPC Service User:/home/admin:/bin/bash`，该用户的登录 Shell 是 `/bin/bash`，属于**可登录的普通用户**。

### 总结

- **系统用户**：约 20 + 个（用于支撑系统服务，无需关注）；
- **可登录的普通用户**：仅 1 个（`admin`）。

日常操作中，你可以用 `admin` 用户进行博客部署、文件管理等工作，如需系统级权限可通过 `sudo` 临时提权。
