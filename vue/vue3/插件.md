## vue-native-websocket 插件使用关键点步骤

基于项目中的实际实现，以下是该插件的核心使用流程和关键点：

------

### **步骤 1：安装依赖**



```
npm install vue-native-websocket --save
```

------

### **步骤 2：在 main.js 中导入并配置**

```
import Vue from 'vue';
import websocket from 'vue-native-websocket';

// 1. 构建 WebSocket 连接地址
const url = `${window.location.protocol === 'https:' ? 'wss' : 'ws'}:${window.location.host}${process.env.VUE_APP_BASE_API}`;

// 2. 使用 Vue.use() 安装插件
Vue.use(websocket, url + '/websocket', {
  store: store,                    // 关键：传入 Vuex store，自动触发 mutation
  format: 'json',                  // 消息格式，自动解析为 JSON
  reconnection: true,              // 自动重连
  reconnectionAttempts: Infinity,  // 重连次数（无限）
  reconnectionDelay: 2000,         // 重连间隔（毫秒）
});
```

**配置参数说明：**

| 参数                   | 说明                                    | 必填   |
| ---------------------- | --------------------------------------- | ------ |
| `store`                | Vuex store 实例，用于自动触发 mutation  | **是** |
| `format`               | 消息格式（`json`/`text`/`arraybuffer`） | 否     |
| `reconnection`         | 是否自动重连                            | 否     |
| `reconnectionAttempts` | 最大重连次数                            | 否     |
| `reconnectionDelay`    | 重连间隔时间（ms）                      | 否     |

------

### *步骤 3：在 Vuex Store 中定义 SOCKET_ Mutation**

插件会**自动**根据 WebSocket 事件触发对应的 mutation：

```
const store = new Vuex.Store({
  state: {
    socket: {
      isConnected: false,
      message: '',
      reconnectError: false,
    },
    websocketMessage: '',
    // ... 其他业务数据
  },
  mutations: {
    // 连接成功
    SOCKET_ONOPEN(state, event) {
      Vue.prototype.$socket = event.currentTarget;
      state.socket.isConnected = true;
    },
    // 连接关闭
    SOCKET_ONCLOSE(state, event) {
      state.socket.isConnected = false;
    },
    // 连接错误
    SOCKET_ONERROR(state, event) {
      console.error('WebSocket Error:', event);
    },
    // 收到消息（核心）
    SOCKET_ONMESSAGE(state, message) {
      state.socket.message = message;
      // 根据消息类型分发处理
      if (message.type === 'getSysDevInfo') {
        this.commit('setSysDevInfoMessage', message);
      }
    },
    // 重连中
    SOCKET_RECONNECT(state, count) {
      console.info('Reconnecting...', count);
    },
    // 重连失败
    SOCKET_RECONNECT_ERROR(state) {
      state.socket.reconnectError = true;
    },
    // 自定义数据存储 mutation
    setSysDevInfoMessage(state, data) {
      state.websocketMessage = data;
    },
  },
});
```

------

### **步骤 4：在组件中使用**

#### **方式 1：监听连接状态**

```
<template>
  <div>
    <span v-if="$store.state.socket.isConnected">已连接</span>
    <span v-else>未连接</span>
  </div>
</template>
```

#### **方式 2：获取实时数据**

```
import { mapState } from 'vuex';

export default {
  computed: {
    ...mapState(['websocketMessage']),
  },
  watch: {
    websocketMessage(newVal) {
      // 数据更新时触发业务逻辑
      console.log('收到新消息:', newVal);
    },
  },
};
```

#### **方式 3：主动发送消息**

```
// 通过 Vue.prototype.$socket 发送消息
this.$socket.send(JSON.stringify({ type: 'ping', data: 'hello' }));
```

------

### **步骤 5：消息处理流程**

```
┌─────────────────────────────────────────────────────────────┐
│  后端推送消息 → WebSocket 接收 → 触发 SOCKET_ONMESSAGE      │
│                                         ↓                   │
│                              根据 message.type 分发         │
│                                         ↓                   │
│                    commit('setXXXMessage', data)           │
│                                         ↓                   │
│                          更新 Vuex State                    │
│                                         ↓                   │
│                      组件响应式更新视图                      │
└─────────────────────────────────────────────────────────────┘
```

------

### **关键注意事项**

| 关键点           | 说明                                                       |
| ---------------- | ---------------------------------------------------------- |
| **自动触发机制** | 插件会自动调用 `SOCKET_*` 命名的 mutation，无需手动调用    |
| **消息格式**     | 配置 `format: 'json'` 后，收到的消息会自动解析为 JSON 对象 |
| **重连策略**     | 建议设置 `reconnection: true` 保证连接稳定性               |
| **错误处理**     | 通过 `SOCKET_ONERROR` 和 `SOCKET_RECONNECT_ERROR` 处理异常 |
| **全局访问**     | 通过 `Vue.prototype.$socket` 可在任意组件中发送消息        |

------

### **完整流程图**

```
main.js 配置
    ↓
Vue.use(websocket, url, { store, format, reconnection })
    ↓
WebSocket 连接建立
    ↓
自动触发 SOCKET_ONOPEN
    ↓
后端推送消息 → SOCKET_ONMESSAGE
    ↓
根据 type 分发到对应 mutation
    ↓
更新 state → 组件响应式更新
```
